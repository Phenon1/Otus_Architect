# Endpoint приема сообщений Агента

`MessageBrokerProj` содержит RabbitMQ endpoint, который принимает команды от приложений Агентов и маршрутизирует их в очередь команд нужной игры.

## Входящее сообщение

Endpoint читает JSON-сообщения из очереди `agent.commands`.

Контракт сообщения:

| Поле | Тип | Обязательное | Назначение |
|---|---|---|---|
| `gameId` | `string` | да | Идентификатор игры, в очередь которой нужно передать сообщение. |
| `objectId` | `string` | да | Идентификатор игрового объекта внутри игры. |
| `operationId` | `string` | да | Публичный идентификатор операции; он не используется как IoC-ключ. |
| `args` | `object` | да | Параметры операции; структура зависит от конкретной операции. |
| `timestamp` | `DateTime` | нет | Время создания сообщения. |
| `version` | `string` | нет | Версия контракта сообщения. |

`timestamp` и `version` имеют значения по умолчанию на стороне DTO, но Агент может передавать их явно.

Формат не зависит от конкретных классов команд. Новое правило игры добавляется регистрацией разрешенной фабрики операции в IoC-scope игры. Endpoint и `InterpretCommand` при этом не меняются.

## Поток обработки

1. `RabbitMqGameMessageEndpoint` десериализует тело RabbitMQ-сообщения в `GameMessage`.
2. Endpoint находит нужный `GameContext` по `gameId` через `IGameRegistry`.
3. Endpoint создает `InterpretCommand` и кладет ее в `QueueICommand` выбранной игры.
4. `InterpretCommand` выполняется в процессоре команд игры, переключается на IoC-scope игры, проверяет сообщение через `game.message.validate`, получает разрешенную фабрику команды через `game.operations.allowed`, создает доменную команду и ставит ее в очередь через `game.commands.enqueue`.

`operationId` проходит через белый список IoC-регистраций и никогда не передается в `IoC.Resolve` как имя зависимости. Агент не может прислать произвольное внутреннее имя команды.

## Ответ

Если в RabbitMQ-сообщении задан `ReplyTo`, endpoint публикует `GameResponse` в эту очередь и копирует входящий `CorrelationId`.

Контракт ответа:

| Поле | Тип | Обязательное | Назначение |
|---|---|---|---|
| `success` | `bool` | да | Признак успешного приема сообщения endpoint-ом. |
| `error` | `string \| null` | нет | Текст ошибки, если сообщение не принято. |
| `gameState` | `object \| null` | нет | Состояние игры; в этом endpoint не заполняется. |
| `timestamp` | `DateTime` | да | Время формирования ответа. |

`gameState` в рамках этого endpoint не заполняется, потому что endpoint только принимает и маршрутизирует команды.

## Пример
```json
{
  "gameId": "game-1",
  "objectId": "ship-1",
  "operationId": "startMove",
  "args": {
    "velocity": 2
  },
  "timestamp": "2026-06-16T00:00:00Z",
  "version": "1.0"
}
```

В консоли endpoint должно появиться сообщение о принятой команде.
